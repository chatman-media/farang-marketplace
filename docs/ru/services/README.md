# 🔧 Сервисы Thailand Marketplace

Данная папка содержит подробную документацию по всем микросервисам платформы
Thailand Marketplace. Каждый сервис имеет свою собственную документацию с
детальным описанием архитектуры, API и функциональности.

## 📋 Список сервисов

### 🚪 API Gateway

**Порт разработки:** 3000
**Описание:** Централизованный шлюз для всех API запросов
**Основные функции:**

- Маршрутизация запросов к микросервисам
- Rate limiting и защита
- Централизованное логирование (Pino)
- CORS и middleware управление
- Health checks для всех сервисов

### 👤 [User Service](./user-service.md)

**Порт разработки:** 3001
**Описание:** Сервис аутентификации и управления пользователями
**Основные функции:**

- Регистрация и аутентификация пользователей
- Управление профилями пользователей
- OAuth 2.0 интеграция (Google, Apple, TikTok, Telegram, LINE, WhatsApp)
- JWT токены и сессии
- Управление ролями и разрешениями

### 🏠 [Listing Service](./listing-service.md)

**Порт разработки:** 3003
**Описание:** Управление объявлениями маркетплейса
**Основные функции:**

- CRUD операции с объявлениями
- Поиск и фильтрация
- Геолокационный поиск
- Управление категориями
- Управление изображениями (WebP оптимизация)

### 📅 [Booking Service](./booking-service.md)

**Порт разработки:** 3004
**Описание:** Система бронирования и управления транзакциями
**Основные функции:**

- Создание и управление бронированиями
- Календарь доступности
- Статусы бронирований
- Интеграция с платежной системой
- Email и Telegram уведомления

### 💳 [Payment Service](./payment-service.md)

**Порт разработки:** 3009
**Описание:** Интеграция платежей с TON blockchain и традиционными методами
**Основные функции:**

- TON blockchain платежи
- Stripe для карт
- PromptPay для Таиланда
- Webhook обработка
- Возвраты и споры

### 📞 [CRM Service](./crm-service.md)

**Порт разработки:** 3007
**Описание:** Многоканальные коммуникации и управление клиентами
**Основные функции:**

- Управление лидами и клиентами
- Email, Telegram, WhatsApp, LINE интеграция
- Автоматизированные кампании
- Аналитика взаимодействий
- Система оценки лидов

## 🏗️ Архитектура микросервисов

Все сервисы построены по принципу микросервисной архитектуры:

- **Независимое развертывание**: Каждый сервис может быть развернут независимо
- **Собственная база данных**: Каждый сервис имеет свою PostgreSQL базу данных
- **API Gateway**: Централизованная точка входа для всех запросов
- **Service Discovery**: Автоматическое обнаружение сервисов
- **Load Balancing**: Распределение нагрузки между экземплярами
- **Health Checks**: Мониторинг состояния сервисов

## 🔗 Взаимодействие сервисов

```
API Gateway (3000)
    ├── User Service (3001) - аутентификация
    ├── Listing Service (3003) - объявления
    ├── Booking Service (3004) - бронирования
    │   └── → Payment Service (3009) - платежи
    └── CRM Service (3007) - коммуникации
```

## 🛠️ Общие технологии

- **Runtime**: Node.js 20+ + TypeScript
- **Framework**: Fastify 5.x
- **Database**: PostgreSQL + Drizzle ORM
- **Cache**: Redis
- **Testing**: Vitest
- **Package Manager**: Bun
- **Logging**: Pino
- **Monorepo**: Turborepo
- **Linting**: Biome
- **Validation**: Zod schemas
- **Authentication**: JWT + Fastify plugins

- Улучшена обработка ошибок

- Оптимизированы middleware для аутентификации

## 🚀 Быстрый старт

Для запуска всех сервисов в режиме разработки:

```bash
# Установка зависимостей
bun install

# Запуск всех сервисов с UI панелью Turbo
bun run dev:ui

# Запуск всех сервисов без UI
bun run dev

# Запуск конкретного сервиса
cd services/user-service && bun run dev
```

## 📝 Соглашения

- Все API используют REST архитектуру
- Стандартные HTTP коды ответов
- JSON формат для всех запросов/ответов
- JWT токены для аутентификации
- Единый формат ошибок
- Централизованное логирование через Pino
- Drizzle ORM для работы с базой данных
- Zod для валидации данных

## 📱 Telegram Integration

Все сервисы интегрированы с Telegram для уведомлений и коммуникации:

- **Booking Service** → уведомления о новых бронированиях
- **Payment Service** → уведомления об оплатах и статусах транзакций
- **CRM Service** → многоканальная коммуникация (Telegram, WhatsApp, Email, LINE)
- **User Service** → Telegram Login Widget для аутентификации

### Планируется:

- **Telegram Bot** - бот для уведомлений и базовых операций
- **Telegram Mini App** - полноценный маркетплейс внутри Telegram
- Интеграция TON платежей в Telegram Mini App
